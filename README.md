##**1. Основна дія**
Рух платформи вліво та вправо, щоб відбивати м'яч і руйнувати цеглини, уникаючи його падіння за нижню межу екрана.


**2. Пристрій керування**
Схема: Дві кнопки.

Опис: Гравець використовує лише дві клавіші (наприклад, A та D або стрілки вліво/вправо) для миттєвого горизонтального переміщення платформи (Paddle). Ніяких додаткових кнопок для стрибків чи стрільби не потрібно, що максимально фокусує увагу на таймінгах.


**3. Схема рівня**

Простір руху: Прямокутне ігрове поле, обмежене невидимими стінами з трьох сторін (зліва, зверху, справа).

Старт гравця (Player): Горизонтальна платформа (Paddle) внизу по центру.

Об'єкт дії (Ball): М'яч, що стартує трохи вище платформи.

Перешкоди/Цілі (Bricks): Сітка з 50 прямокутників (10х5) у верхній половині екрана.

Небезпека (Death Zone): Нижня відкрита межа екрана (координата Y < -5.5), куди може впасти м'яч.


**4. Ускладнення дії за формулою “Тиск — Обмеження — Наслідок”**
Що створює напругу (Тиск)?

Прискорення ритму / Постійний рух: М'яч ніколи не зупиняється і має фіксовану високу швидкість (maxVelocity = 15), що вимагає від гравця постійної концентрації та швидкої реакції.

Що саме обмежується?

Кількість спроб: У гравця є лише 5 життів (сердечок) на всю ігрову сесію.

Наслідок помилки (Outcome):

Втрата ресурсу: Якщо м'яч падає за нижню межу, гравець втрачає одне життя (графічна іконка зникає), а м'яч повертається в центр. Якщо життів стає 0 — гра закінчується.


**5. Асети для гри**
Спрайти оточення та персонажа: Базові примітиви — квадрат (Square) для платформи та цеглин, коло (Circle) для м'яча. Для цеглин використовується процедурний градієнт.

UI (Інтерфейс): Текстовий шрифт TextMeshPro для відображення рахунку (Score) у правому верхньому куті та UI-панель "Game Over".

Звукові ефекти (планується): Короткий "піп" при ударі об стіну/падл, дзвінкий звук скла при руйнуванні цеглини, низький гудок при втраті життя.


**6 & 7. Монетизаційна модель (Епоха ранніх аркад)**
Вартість сесії: 25 центів за 1 гру (яка включає 5 життів).

Мотивація гравця: Основна рушійна сила — це набрати найбільшу кількість очок (High Score), руйнуючи блоки (кожен блок дає +10 очок) та повністю "зачистити" екран.

“One more try” loop: Коли гравець втрачає останнє (п'яте) життя, гра показує екран "Game Over" і через 2 секунди автоматично перезавантажує рівень. Ця миттєва готовність гри до нового раунду створює сильний психологічний ефект "ну зараз точно вийде", змушуючи гравця кидати ще одну монетку.


**8. Збірка сцени (Логіка всередині скриптів)**
Сцена зібрана в Unity з реалізацією ручної логіки:

Створено скрипт PlayerMovement, який зчитує ввід гравця та рухає платформу по осі X.

Створено скрипт BouncyBall, який відстежує зіткнення (OnCollisionEnter2D) з об'єктами за тегом "Brick", знищує їх (Destroy), додає очки та оновлює текстовий індикатор scoreTxt.

Реалізовано систему життів (масив GameObject[] livesImage), яка по черзі вимикає UI-іконки при падінні м'яча нижче визначеної координати Y.

Використано Physics Material 2D (Bounciness 1, Friction 0) для ідеального відскоку м'яча без втрати енергії.


**9. Концепт-документ гри**
Назва гри: Neon Brick Destroyer.

Основна дія (1 речення): Рух платформи вліво/вправо для відбивання м'яча та руйнування цеглин.

Схема керування: Дві кнопки (горизонтальні осі).

Мета гри: Знищити всі цеглини на рівні та набрати максимум очок.

Правила: Гравець має 5 життів. Падіння м'яча = -1 життя. Знищення цеглини = +10 очок.

Тиск — Обмеження — Наслідок: Висока швидкість м'яча — Ліміт у 5 спроб — Втрата життя/Кінець гри.

Core Loop: Відбиття м'яча -> Руйнування перешкоди -> Оновлення рахунку -> Уникнення падіння.

Стиль і атмосфера: Мінімалістичний 2D-аркадний стиль із яскравими градієнтними кольорами на темному тлі.

Монетизаційна модель: 25¢ за одну сесію (5 життів), фокус на бажанні перевершити власний рекорд.
